# CLI 字幕评分调试工具 — 扣分项定位到具体字幕行

## 问题

当前评分系统返回汇总扣分（如"时间轴重叠(3处) -15"），用户无法知道哪 3 条字幕触发了重叠，也无法验证 last_end_ms 的计算逻辑。用户反馈评分不合理时，缺乏具体的定位信息。

## 功能设计

### CLI 命令

```bash
thunder-subtitle review --debug <字幕文件路径> [--nfo <nfo路径>]
```

不标记任何审核状态，仅输出详细评分报告。

### 条目数诊断输出项

| 项目 | 说明 |
|------|------|
| 文件大小 | 快速判断是否空文件 |
| 有效行数 | 排除空行的总行数 |
| 正则匹配命中 | `_SRT_PATTERN.finditer` 命中次数 |
| 未匹配尾部 | 最后一条匹配之后还剩多少字节未消费 |
| 原始文本预览 | 前 500 字符 + 后 200 字符，方便判断格式 |

### 改动范围

| 文件 | 改动 |
|------|------|
| `thunder-subtitle-py/commands/review.py` | 新增 `--debug` 参数 + `cmd_review_debug` 函数 + 条目数诊断 |
| `thunder-subtitle-py/cli.py` | 注册 `review --debug` 子命令 |
| `thunder-subtitle-py/src/reviewer/_srt.py` | `_check_srt_quality` 返回详细条目列表（含行号和内容）；`_parse_srt_entries` 返回未匹配信息 |
| `thunder-subtitle-py/src/reviewer/_review.py` | `ReviewItem` 新增 `detail_entries: list[dict]`；暴露文件原始内容供诊断 |

### 核心改造

`_check_srt_quality` 原本只返回 `ai_flags: list[str]`。增加 debug 模式时返回结构化详情：

```python
@dataclass
class SrtIssue:
    type: str           # "overlap" | "inverted_ts" | "gap" | "fast_read" | ...
    entry_index: int     # 第几条字幕 (1-based)
    start_ms: int
    end_ms: int
    content: str         # 字幕文本（截断至 40 字）
    detail: str = ""     # 额外描述（如重叠的对象、阅读速度数值）

# _check_srt_quality 新增可选返回值
-> tuple[list[str], list[SrtIssue]]  # (ai_flags, issues)
```

## Acceptance Criteria

- [ ] `review --debug <文件>` 输出完整评分报告
- [ ] 每个扣分项标注条目序号 + 原始文件行号 + 时间戳
- [ ] 显示 last_end_ms 的逆向扫描过程（跳过了哪些条目、为什么跳过）
- [ ] 正常模式 `review` 不受影响（不产生额外开销）
- [ ] 条目数诊断：显示文件大小、行数、正则命中数、未匹配尾部、原始文本预览
- [ ] 1/1 异常情况能输出原始文本帮助判断格式问题
- [ ] `ruff check` 零错误

## Out of Scope

- Web UI 集成（仅 CLI）
- 自动评分校准建议
- 批量调试模式
