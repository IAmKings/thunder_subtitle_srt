# 验证模块 UX 重构：电影 → 字幕两级选择流

## Goal

将验证页从"平铺所有字幕文件"改为"电影 → 字幕"两级选择流，选中电影后进入专注的审核模式。

## What I already know

### 当前数据模型

```typescript
ReviewItem {
  file_path: string;   // 电影目录路径 (e.g. /media/movies/流浪地球)
  file_name: string;   // 字幕文件名 (e.g. 流浪地球.zh.srt)
  score: number;       // 质量评分 0-100
  size_bytes: number;  // 文件大小
  chinese_ratio: number;
  encoding: string;
  review_status: "ok" | "fail" | "not_reviewed";
}
```

### 当前问题

1. 扁平列表：所有电影的字幕混在一起，无法按电影组织
2. 选中后无上下文切换，用户不知道在看哪个电影的字幕
3. 筛选/排序/搜索混杂所有电影数据，无意义

## Requirements

### R1: 两级选择流

**第一级（电影列表）**：
- 标题显示「等待验证」
- 搜索框模糊匹配电影名（多电影时找电影是核心需求）
- 列表按电影分组，显示：电影名 + 字幕数量

**第二级（审核中）**：
- 标题变为「审核中 - {电影名}」，左侧加返回箭头
- 搜索框消失，改为状态筛选 chip [全部][未标][✓][✗] + 大小排序
- 列表仅显示该电影的字幕文件
- 选中字幕高亮（左侧粗边框）

### R2: 搜索 / 筛选按阶段切换

- **电影列表**：搜索框模糊匹配电影名，无状态筛选 chip
- **审核中**：状态筛选 chip + 大小排序，无搜索框

### R3: 批量操作优化

- 选中电影后，按钮改为「全部通过」+「全部失败」
- 操作仅影响当前电影的字幕

## Acceptance Criteria

* [ ] 电影列表：搜索框模糊匹配电影名，无状态筛选
* [ ] 点击电影 → 审核中：搜索框消失，显示状态筛选 chip + 大小排序
* [ ] 标题切换：「等待验证」→「审核中 - {电影名}」+ 返回箭头
* [ ] 点击返回箭头 → 回到电影列表，清除选中
* [ ] 选中字幕高亮明显（左侧粗边框 + 背景色差）
* [ ] 批量操作针对当前电影的字幕

## Out of Scope

* 电影组的展开/折叠动画
* 跨电影字幕对比
* 审核历史记录
